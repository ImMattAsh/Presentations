Basic Concepts

Projects

  • Typically tied to a solution (.sln)
  • Contains child projects -- and/or -- build configurations
  • They are what appears on your home screen

Basic Concepts

Project > Build Configurations

  • High level container for your build process
  • Contains build steps
  • Can have dependencies or be another configuration's dependent configuration (we'll get to dependencies)

Basic Concepts

... > Build Configurations > Build Steps

  • Task for a build agent to execute
  • Contains build steps
  • Can have dependencies or be another confiugrations dependent configuration

Octopus Deploy

Basic Concepts

Basic Concepts

Octopus Server vs. Tentacles

Basic Concepts

Basic Concepts

Projects

  • Defines a process for deployment
  • Contains variables
  • Has many releases
  • Is associated with a life cycle

Basic Concepts

Project > Releases

  • Contains a deployable package
  • Can have release notes (automate via a build feature)
  • Has a version number
  • Can be deployed/promoted to various environments

Basic Concepts

Project > Variables

  • *.config AppSettings
  • *.config ConnectionStrings
  • Powershell Scripts
  • Scoping
    • Environments
    • Machines
    • Roles
    • Deployment Steps
    • The most specific scope will be used
  • Library Sets (we'll get to it)

Basic Concepts

Project > Deployment Process Steps

  • Deploy a NuGet package
  • Run a Powershell script
  • Send an email
  • Manual Intervention
  • Deploy to Windows Azure
  • and many more...
  • Scoping (similar to variables)
  • Check out the octopus library for custom templates: https://library.octopusdeploy.com/#!/listing

Basic Concepts

Octopack and TeamCity Plugins

  • Octopack
    • Generates an Octopus-style NuGet package
    • Add to any VS project that is to be deployed
    • Can be found on nuget.org
  • TeamCity Plugin
    • Adds 3 runners to your Build Step options: Create/Deploy/Promote releases
    • Fires off tasks via the Octopus API

Octopus

"Advanced" Concepts

"Advanced" Concepts

Library Sets

  • Variables outside of Projects
  • Can be included on many projects
  • Get applied the same as project variables

"Advanced" Concepts

Step Templates

  • A template for a reusable and custom Deployment Process Step
  • Can be included on many projects

"Advanced" Concepts

External Feeds

  • NOTE: Octopus has its own internal feed, however, it's not externally accessible
  • Add any nuget repositories you want. In our case, I've already added our TeamCity NuGet feed which is externally accessible

"Advanced" Concepts

Script Modules

  • Collections of PowerShell functions and cmdlets that can be included in multiple projects
